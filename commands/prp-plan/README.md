# /ecc:prp-plan

分析代码库并生成可单次执行的完整功能实现计划；计划优先进入项目现有的 `docs/` 文档体系。

---

## 功能

- 解析功能描述、参考文件或 PRD 中下一个可执行阶段
- 搜索相似实现、命名、错误处理、日志、类型、测试、配置与依赖模式
- 记录入口、数据流、状态变更、接口契约和架构约定
- 输出自包含的实施步骤、测试策略、验证命令、验收标准与风险
- 项目根目录存在 `docs/` 时，将计划保存到 `docs/plan/`
- 项目没有 `docs/` 时，回退保存到 `.claude/PRPs/plans/`

## 用法

```text
/ecc:prp-plan <功能描述 | path/to/prd.md>
```

## 执行流程

1. 检测输入是 PRD、参考文件还是自由文本
2. 解析需求并进行歧义门禁与复杂度评估
3. 深入分析代码库模式与关键调用链
4. 按需查阅外部官方资料
5. 设计范围、方案、UX 变化与明确的非目标
6. 根据项目根目录是否存在 `docs/` 选择唯一输出目录
7. 生成计划；若来源是 PRD，则同步阶段状态与计划路径

## 约束

- `docs/` 检测仅针对项目根目录
- 存在 `docs/` 时固定使用 `docs/plan/`，不存在时才使用 `.claude/PRPs/plans/`
- 只写一份计划，不在两个目录重复生成
- 计划必须包含真实代码示例、文件路径和行号，不得臆造代码库模式
- 实施者应仅依赖计划即可完成工作，无需再次搜索代码库或补问上下文
