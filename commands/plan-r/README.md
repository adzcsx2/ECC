# /ecc:plan-r

规划 → 执行 → 代码审查，三阶段强制流水线。用户只需确认计划，后续自动完成执行和审查。

---

## 功能

- 需要完整走完规划、执行、代码审查三个环节时使用
- 确保每次实现都经过代码审查，避免遗漏安全或质量问题
- 适合中等规模功能开发、Bug 修复、重构任务

## 用法

```
/ecc:plan-r <任务描述>
```

示例：

```
/ecc:plan-r 给用户表加登录时间字段
/ecc:plan-r 修复订单状态不同步的 bug
```

## 执行流程

1. **Phase 1 — 规划**：调用 `/ecc:plan` 技能生成分步计划，等待用户确认（必须包含"确认"/"yes"等关键词）
2. **Phase 2 — 执行**：主 Agent 按计划逐步实现，不跳过任何步骤
3. **Phase 3 — 代码审查**：调用 `code-reviewer` 子代理审查所有变更，自动修复 CRITICAL 问题

## 约束

- 未收到用户确认不得进入 Phase 2
- Phase 3 必须通过真实的 Agent tool 调用，不得口头声明"已完成审查"
- 任意 Phase 为 ❌ 时命令视为失败，需补做
