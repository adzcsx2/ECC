# /ecc:prp-prd

通过分阶段提问、市场与代码库调研生成可验证的产品需求文档；优先进入项目现有的 `docs/product/` 文档体系。

---

## 功能

- 从问题、用户、证据和成功指标出发澄清产品需求
- 结合市场资料与现有代码评估可行性和风险
- 记录 MVP、非目标、开放问题、决策和实施阶段
- 项目根目录存在 `docs/` 时，将 PRD 保存到 `docs/product/`
- 项目存在 `docs/` 时使用中文产品文档文件名
- 项目没有 `docs/` 时，回退保存到 `.claude/PRPs/prds/`

## 用法

```text
/ecc:prp-prd <产品或功能想法>
```

## 执行流程

1. 确认核心问题和目标用户
2. 收集基础证据与成功标准
3. 调研市场、竞品和现有代码
4. 澄清愿景、用户场景与约束
5. 评估技术可行性和风险
6. 确认 MVP、必须项、非目标和开放问题
7. 根据项目是否存在 `docs/` 选择唯一输出目录
8. 生成 PRD 并使用真实路径交给 `/ecc:prp-plan`

## 约束

- `docs/` 检测只针对项目根目录
- 存在 `docs/` 时固定使用 `docs/product/`，文件名采用 `{中文产品或功能名称}需求文档.md`
- 不存在 `docs/` 时才使用 `.claude/PRPs/prds/{kebab-case-name}.prd.md`
- 只生成一份 PRD，不在主目录和回退目录重复写入
- 输出前必须读取项目文档规则和现有产品文档命名模式
- 缺少事实时写 `TBD - needs research`，不得用合理猜测填充需求
