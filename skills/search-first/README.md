# /ecc:search-first

编码前先研究的标准化工作流：搜索现有工具、库和模式，避免重复造轮子。

---

## 功能

- 工具可用性预检：在依赖搜索渠道之前先验证其可用性
- 并行搜索：同时搜索包管理器（npm/PyPI）、MCP 服务器、GitHub 和 web
- 评估与决策矩阵：根据匹配度、维护状态、社区活跃度等评分，决定采用/扩展/组合/自建
- 快捷搜索分类：按开发工具、AI/LLM 集成、数据与 API、内容与发布分类提供搜索参考

## 用法

- `/ecc:search-first` - 开始新功能开发前、添加依赖或集成时自动激活
- 快模式：在编码前自我检查 4 个搜索渠道
- 完整模式：启动 researcher 代理执行结构化搜索

## 适用场景

- 开始实现一个新功能前，检查是否有现成的库或 MCP 可复用
- 在编写自定义工具类、辅助函数或抽象层之前
- 避免安装过大的包来实现一个小功能（依赖膨胀）
- 与 planner 和 architect 代理配合，在架构设计阶段就纳入现有工具
