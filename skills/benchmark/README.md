# /ecc:benchmark

性能基准测量与回归检测：页面、API、构建和前后对比。

---

## 功能

- 页面性能模式：测量 Core Web Vitals（LCP、CLS、INP、FCP、TTFB）、资源大小、网络请求数
- API 性能模式：对端点进行 100 次请求采样，测量 p50/p95/p99 延迟
- 构建性能模式：冷构建时间、热更新速度、测试套件耗时、类型检查时间
- 前后对比模式：保存基线 → 做变更 → 对比增量，检测性能回归
- 对标 SLA 目标输出 verdict（PASS/WARN/FAIL）

## 用法

- `/ecc:benchmark` - 测量性能基线、检测 PR 前后的回归、对比技术栈方案

## 适用场景

- PR 前后测量性能影响
- 为项目设置性能基线
- 用户反馈"感觉慢"时排查
- 发布前确保满足性能目标
- 对比你的技术栈与其他方案
