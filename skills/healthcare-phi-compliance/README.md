# /ecc:healthcare-phi-compliance

受保护健康信息合规模式：数据分类、访问控制、审计追踪与常见泄漏防范。

---

## 功能

- 数据分类：PHI（受保护健康信息）和 PII（非患者敏感数据）精确定义与区分
- 行级安全访问控制：基于设施的 RLS 策略、审计日志仅可插入不可修改/删除
- 审计追踪：每次 PHI 访问/修改必须记录操作者、患者、操作类型、变更前后数据
- 常见泄漏向量识别：错误消息、控制台输出、URL 参数、浏览器存储、服务角色密钥
- 数据库模式标记：在列级别通过 COMMENT 标记 PHI/PII 字段

## 用法

- `/ecc:healthcare-phi-compliance` - 实现患者数据保护，包括数据分类、行级安全、审计日志和泄漏防范

## 适用场景

- 构建任何涉及患者记录的功能
- 实现临床系统的访问控制或认证
- 设计医疗数据数据库模式
- 构建返回患者或临床数据的 API
- 实现审计追踪或日志
- 审查代码的数据暴露漏洞
