# /ecc:logistics-exception-management

货运异常处理专业方案：延迟、损坏、丢失、争议的解决流程、索赔程序和承运商策略。

---

## 功能

- 异常分类体系（延迟、损坏、短缺、拒收、丢失等 11 种类型）
- 按运输模式（LTL、FTL、包裹、多式联运、海运、空运）的承运商行为策略
- 索赔流程（Carmack 法案、文件要求、响应时限）
- 严重性分级和资金阈值决策框架（吸收 vs 争取索赔）
- 季节性规律、欺诈识别和多异常并发优先级排序
- 客服和内部沟通模板与升级链

## 用法

- `/ecc:logistics-exception-management` - 处理货运异常、索赔和承运商争议

## 适用场景

- 货物延迟、损坏、丢失或被拒收
- 承运商责任争议、附加费或滞留费纠纷
- 客户因错过交货窗口而投诉升级
- 向承运商或保险公司提交货运索赔
- 构建异常处理 SOP 或升级协议
