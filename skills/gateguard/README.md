# /ecc:gateguard

编辑前强制事实调查的 PreToolUse 门禁钩子，阻止凭猜测修改文件。

---

## 功能

- 在首次 Edit/Write/Bash 操作前阻止并强制进行事实调查（文件引用、函数影响、数据模式等）
- 三阶段门禁：禁止 -> 强制收集事实 -> 允许重试
- 不支持自我评估方式，强制通过实际调查创建上下文感知
- 实验验证可提升输出质量 +2.25 分（对比无门禁智能体）
- 支持 MultiEdit 批量文件各自门禁、破坏性 Bash 命令每次触发

## 用法

- `/ecc:gateguard` - 启用事实强制门禁，在编辑文件前自动调查影响范围和数据模式。可通过 `ECC_GATEGUARD=off` 关闭

## 适用场景

- 文件编辑影响多个模块的代码库工作
- 使用特定数据模式或日期格式的项目
- 需要 AI 生成代码匹配现有模式的团队
- Claude 倾向于猜测而非调查的任何工作流
