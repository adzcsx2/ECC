# /ecc:golang-testing

Go 测试模式：表驱动测试、基准测试、模糊测试及 TDD 方法论。

---

## 功能

- TDD 工作流：RED-GREEN-REFACTOR 循环在 Go 中的完整实践
- 表驱动测试：覆盖多场景的正向和错误路径测试
- 测试辅助函数：t.Helper()、临时目录、断言辅助、Golden 文件模式
- 基于接口的 Mock：定义接口依赖并在测试中替换为 Mock 实现
- 基准测试与模糊测试：性能基准、内存分配分析、模糊测试覆盖范围

## 用法

- `/ecc:golang-testing` - 编写 Go 测试，包括 TDD 循环、表驱动测试、Mock、基准测试和模糊测试

## 适用场景

- 编写新的 Go 函数或方法
- 为现有代码添加测试覆盖率
- 为性能关键代码创建基准测试
- 实现输入验证的模糊测试
- 在 Go 项目中遵循 TDD 工作流
