# /ecc:email-ops

邮箱分类、起草、发送验证与已发送邮件跟进的操作工作流。

---

## 功能

- 邮箱分类（triage）：清理低价值邮件，识别高优先级线程
- 邮件起草（draft）：基于线程历史、品牌语气和上下文撰写回复或新邮件
- 发送验证（send verification）：通过已发送文件夹或客户端确认邮件确实送达
- 状态报告：明确标注 drafted / approval-pending / sent / blocked / awaiting verification
- 账户与线程定位：在操作前确认使用的邮箱账户和邮件线程

## 用法

- `/ecc:email-ops` - 当需要整理收件箱、起草或发送邮件、确认邮件是否已发送时激活

## 适用场景

- 需要快速分类和清理收件箱中的低价值邮件
- 起草业务邮件回复或新邮件
- 确认某封邮件是否已经成功发送
- 在切换邮箱账户时需要精准定位操作面
- 处理需要跟进确认的邮件线程
