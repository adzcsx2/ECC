# /ecc:production-audit

基于本地证据的生产就绪审计，用于发布前审查、合并后检查和预发布风险评估，不将仓库数据发送到外部审计服务。

---

## 功能

- 从本地和用户授权证据构建审计，不运行未锁定的远程代码
- 检查安全认证、数据完整性、支付 webhook、运维就绪和用户体验
- 按评分等级输出建议：Blocked(0-49)、Risky(50-69)、Launchable(70-84)、Strong(85-100)
- 列出阻塞项、高价值修复项和下一个具体操作
- 证据痕迹清晰可追溯（git 状态、CI、部署 URL）

## 用法

- `/ecc:production-audit` - 当用户问"这能上线吗"、"生产环境中什么会出问题"时使用
- 针对已部署的应用、发布分支或 PR 进行审计

## 适用场景

- 应用发布前检查是否就绪
- 功能合并后的风险检查
- 公开上线、演示或客户部署前的最终审查
- CI 通过但需要生产风险（而非仅测试状态）评估时
