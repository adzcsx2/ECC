# /ecc:workspace-surface-audit

审计工作区的仓库、MCP、插件和环境配置，推荐最优的 ECC 原生技能和自动化方案。

---

## 功能

- 盘点当前工作区已具备的能力：仓库配置、已安装插件、MCP 服务器、环境密钥接口
- 与官方 Claude 插件和本地已安装插件进行对标分析，识别覆盖差距
- 将每个差距映射为正确的 ECC 形态（Skill/Hook/Agent/MCP/连接器）
- 输出五段式审计报告：当前能力、已覆盖领域、仅有原语的差距、缺失集成和优先级建议
- 遵循只读原则，不修改文件，仅提供最高价值的 ECC 原生建设建议

## 用法

- `/ecc:workspace-surface-audit` - 审计当前工作区，获取 ECC 原生能力建设建议

## 适用场景

- 首次设置 Claude Code 或评估当前配置
- 想知道应该安装哪些技能、Hook、MCP 或连接器
- 对比官方市场插件与 ECC 原生覆盖度
- 审查 .env、.mcp.json 和插件设置，发现缺失的工作流层
