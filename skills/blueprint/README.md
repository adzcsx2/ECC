# /ecc:blueprint

将一句话目标转化为多会话、多代理可执行的分步构建计划。

---

## 功能

- 将复杂目标拆分为单 PR 粒度的步骤，每个步骤包含独立上下文摘要
- 依赖图分析，自动检测可并行执行的步骤
- 对抗性审查：由最强模型子代理对计划进行完整性、依赖正确性和反模式检查
- 支持 Git/GitHub 分支/PR/CI 全流程，无 Git 环境时自动降级为直接模式
- 计划变更协议：步骤可拆分、插入、跳过、重排或废弃，保留审计轨迹

## 用法

- `/ecc:blueprint` - 传入目标描述，生成包含依赖关系、并行步骤和审查结果的分步计划文件

## 适用场景

- 需要将大型功能拆分为多个 PR 时
- 跨多个会话的重构或迁移项目
- 需要在多个子代理间协调并行工作时
- 任何上下文丢失会导致返工的长周期任务
