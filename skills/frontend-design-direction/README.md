# /ecc:frontend-design-direction

为前端 UI 工作设定有意图的、产品化的设计方向，避免模板化的生成外观。

---

## 功能

- 编码前选择明确的设计方向：目的、受众、语气、记忆点和约束条件
- 领域匹配：SaaS 工具应密集安静可扫描，作品集/启动页可更富表现力
- 实现指导：优先使用项目既有组件和 token，多维度调色板，CSS 变量保持一致性
- 反模式清单：禁止紫色渐变默认风格、卡片嵌套卡片、单一色调、隐藏产品工作流
- 审查清单：首视口传递产品感、视觉层次支持扫描、排版适配容器、动效面向状态转换

## 用法

- `/ecc:frontend-design-direction` - 在构建或改进网页、仪表盘、落地页等需要更强设计判断的 UI 时激活

## 适用场景

- 构建需要产品化设计感的网页或应用界面
- 改进现有 UI 使其更具辨识度和层次感
- 避免陷入通用模板化外观
- 在团队中统一前端设计方向
- 为不同领域的产品（SaaS、作品集、编辑内容）匹配适当的视觉风格
