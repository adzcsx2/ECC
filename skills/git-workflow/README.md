# /ecc:git-workflow

Git 工作流最佳实践：分支策略、提交规范、合并与变基、冲突解决及团队协作。

---

## 功能

- 三种分支策略详解（GitHub Flow、Trunk-Based、GitFlow）及选型指南
- Conventional Commits 提交消息规范，包含类型、范围、正文模板
- Merge vs Rebase 对比，何时使用哪种及 rebase 禁忌
- Pull Request 完整工作流：标题、描述模板、代码审查清单
- 发布管理：语义化版本、标签创建、变更日志生成

## 用法

- `/ecc:git-workflow` - 获取 Git 工作流指导，包括分支管理、提交规范、PR 流程、冲突解决和发布管理

## 适用场景

- 为新项目设置 Git 工作流
- 决定分支策略
- 编写提交消息和 PR 描述
- 解决合并冲突
- 管理发布和版本标签
- 新团队成员入职培训
