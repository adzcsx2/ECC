# /ecc:backend-patterns

Node.js、Express 和 Next.js 的后端架构模式与最佳实践。

---

## 功能

- 实现 Repository 和 Service 层模式，分离数据访问与业务逻辑
- 设计 REST API 端点：资源命名、查询参数、中间件
- 优化数据库操作：解决 N+1 问题、索引策略、连接池管理
- 集成缓存（Redis、内存缓存、HTTP 缓存头）
- 设置后台任务和异步处理
- 构建中间件（认证、日志、速率限制、验证）

## 用法

- `/ecc:backend-patterns` - 设计和实现可扩展的后端架构

## 适用场景

- 设计 REST 或 GraphQL API 端点
- 实现 Repository、Service 或 Controller 层
- 优化数据库查询
- 添加缓存策略
- 设置后台任务或异步处理
- 构建中间件
