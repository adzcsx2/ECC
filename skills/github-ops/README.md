# /ecc:github-ops

GitHub 仓库运营自动化：Issue 分类、PR 管理、CI/CD 运维和安全监控。

---

## 功能

- Issue 分类工作流：按类型和优先级分类、去重检测、标签管理、自动回复
- PR 管理：CI 状态检查、合并就绪评估、过期 PR 标记、社区 PR 审核
- CI/CD 运维：失败工作流排查、日志分析、flake 测试识别
- 发布管理：从合并的 PR 生成变更日志、创建发布和预发布
- 安全监控：Dependabot 告警检查、密钥扫描、安全依赖自动合并

## 用法

- `/ecc:github-ops` - 管理 GitHub 仓库运营任务，包括 Issue 分类、PR 审核、CI 调试、发布创建和安全监控

## 适用场景

- 分类和管理 Issue（分类、标签、去重、回复）
- 管理 PR 状态（审查状态、CI 检查、过期 PR 处理）
- 调试 CI/CD 失败
- 准备发布和变更日志
- 监控 Dependabot 和安全告警
- 管理开源项目的贡献者体验
