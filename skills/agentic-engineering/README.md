# /ecc:agentic-engineering

以 agent 为主导的工程方法论：评估先行、任务分解、成本感知。

---

## 功能

- 定义完成标准后再执行，避免无目标开发
- 将工作分解为 agent 可独立完成的小单元（15 分钟规则）
- 按任务复杂度路由模型层级：Haiku（简单）→ Sonnet（实现）→ Opus（架构）
- 实施 eval 先行循环：基线评估 → 执行 → 回归评估 → 对比增量
- 追踪每任务的成本指标（模型、token、重试、耗时、成功/失败）

## 用法

- `/ecc:agentic-engineering` - 以 eval 先行、任务分解、成本感知的方式执行工程工作

## 适用场景

- AI agent 承担大部分实现工作的工程团队
- 需要将大型任务拆解为 agent 可消化的单元
- 需要按任务复杂度选择合适的模型
- 需要用量化评估衡量 agent 产出质量
