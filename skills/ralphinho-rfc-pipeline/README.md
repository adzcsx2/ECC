# /ecc:ralphinho-rfc-pipeline

RFC 驱动的多代理 DAG 执行管线，包含质量门禁、合并队列和工作单元编排。

---

## 功能

- 7 阶段管线：RFC 接入、DAG 分解、单元分配、单元实现、单元验证、合并队列集成、系统验证
- 工作单元规格模板（id、依赖、范围、验收测试、风险等级、回滚计划）
- 3 级复杂度分层（隔离编辑、多文件变更、schema/安全/性能变更）
- 每单元质量管线：研究 -> 实现计划 -> 实现 -> 测试 -> 审查 -> 合并就绪报告
- 合并队列规则：依赖失败单元不合并、始终 rebase、每次合并后重跑集成测试
- 停滞单元恢复：踢出活跃队列、快照发现、缩小范围后重试

## 用法

- `/ecc:ralphinho-rfc-pipeline` - 当功能过大无法单代理完成，需拆分为独立可验证的工作单元时使用

## 适用场景

- 需要将大型 RFC 分解为可独立验证的工作单元
- 多代理并行实现需要协调
- 复杂功能需要分阶段、分单元推进
