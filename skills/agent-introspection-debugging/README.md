# /ecc:agent-introspection-debugging

AI agent 故障的结构化自调试工作流。

---

## 功能

- 在盲重试之前捕获故障状态（错误、工具调用序列、上下文压力、环境假设）
- 将故障匹配到已知模式（循环死锁、上下文溢出、连接拒绝、配额耗尽等）
- 应用受控恢复操作：最小可逆修改来验证诊断
- 产出结构化的人类可读调试报告

## 用法

- `/ecc:agent-introspection-debugging` - 对反复失败的 agent 运行进行系统的自调试

## 适用场景

- 达到工具调用上限或循环限制后仍无进展
- 重复重试但没有任何前进
- 上下文膨胀或提示漂移导致输出质量下降
- 文件系统或环境状态与预期不匹配
- 工具故障可能通过诊断和较小修正恢复
