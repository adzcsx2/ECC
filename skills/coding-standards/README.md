# /ecc:coding-standards

跨项目编码规范的基线标准，涵盖命名、不可变性、代码质量和审查准则。

---

## 功能

- 代码质量四原则：可读性优先、KISS、DRY、YAGNI
- TypeScript/JavaScript 命名规范、不可变性模式、错误处理、异步最佳实践
- React 组件结构、自定义 Hooks、状态管理、条件渲染惯用写法
- REST API 设计规范：路由命名、响应格式、Zod 输入验证
- 文件组织结构、注释规范（JSDoc）、性能优化（Memoization、Lazy Loading）

## 用法

- `/ecc:coding-standards` - 获取通用编码规范指导，用于新项目启动或代码审查

## 适用场景

- 启动新项目或新模块时建立代码基线
- 审查代码质量和可维护性
- 强制执行命名、格式化和结构一致性
- 新贡献者编码惯例入门
