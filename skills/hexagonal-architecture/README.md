# /ecc:hexagonal-architecture

六边形架构（端口与适配器）的设计、实现和重构，保持业务逻辑独立于框架和基础设施。

---

## 功能

- 将业务逻辑与传输层、持久化、外部 API 完全解耦
- 通过入站/出站端口抽象，实现同一用例支持多个接口（HTTP、CLI、消息队列）
- 依赖方向始终向内：适配器 -> 应用层 -> 端口接口 -> 领域模型
- 提供 TypeScript/Java/Kotlin/Go 多语言端口与适配器映射方案
- 渐进式迁移策略：按垂直切片逐步提取，避免全量重写

## 用法

- `/ecc:hexagonal-architecture` - 设计或重构为六边形架构，将业务逻辑与框架解耦，支持多种传输方式

## 适用场景

- 构建注重长期可维护性和可测试性的新功能
- 重构领域逻辑与 I/O 混杂的分层或框架耦合代码
- 同一用例支持 HTTP、CLI、消息队列等多接口
- 替换基础设施（数据库、外部 API、消息总线）而不重写业务逻辑
