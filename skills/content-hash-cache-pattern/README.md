# /ecc:content-hash-cache-pattern

使用 SHA-256 内容哈希作为缓存键，实现路径无关、自动失效的文件处理缓存。

---

## 功能

- 内容哈希缓存：基于文件内容（非路径）生成缓存键，文件移动/重命名仍命中缓存
- 冻结数据类：CacheEntry 存储哈希、源路径和处理结果，不可变设计
- O(1) 查找：以 `{hash}.json` 命名的文件缓存存储，无需索引文件
- 服务层分离（SRP）：处理函数保持纯净，缓存逻辑通过包装层附加
- 优雅降级：缓存损坏时自动当作未命中重新处理，永不会崩溃

## 用法

- `/ecc:content-hash-cache-pattern` - 获取基于内容哈希的文件缓存实现模式

## 适用场景

- 构建 PDF 解析、文本提取、图像分析等文件处理流水线
- 处理成本高昂且相同文件会被反复处理
- 需要 `--cache/--no-cache` 命令行选项的 CLI 工具
- 在不修改现有纯函数的前提下添加缓存功能
