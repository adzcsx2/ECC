# /ecc:click-path-audit

追踪 UI 按钮的完整状态变化序列，发现函数各自正常但合并后相互抵消的隐蔽 bug。

---

## 功能

- 六种 bug 模式检测：顺序撤销、异步竞态、闭包过期、缺失状态转换、条件死路径、useEffect 干扰
- 三步执行：映射状态存储 -> 审计每个交互点 -> 生成结构化 bug 报告
- 状态存储副作用映射：标记每个 action 设置的字段和意外重置的字段
- 支持全量审计（分代理并行）、单页审计和存储聚焦审计三种范围
- 每个 bug 输出包含严重程度、模式分类、调用链追踪和具体修复建议

## 用法

- `/ecc:click-path-audit` - 指定目标区域，自动审计所有交互点并报告 bug

## 适用场景

- 系统性调试未发现 bug 但用户反馈按钮不工作时
- 修改 Zustand/Redux store action 后检查所有消费者
- 任何涉及共享状态的重大重构之后
- 关键用户流程发布前的最终验证
