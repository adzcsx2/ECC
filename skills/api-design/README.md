# /ecc:api-design

REST API 设计模式：资源命名、状态码、分页、过滤和错误响应。

---

## 功能

- 设计规范的 RESTful URL 结构（名词复数、kebab-case、嵌套资源）
- 正确使用 HTTP 方法和状态码（GET/POST/PUT/PATCH/DELETE，2xx/4xx/5xx）
- 实现分页、过滤、排序和字段选择
- 设计统一的错误响应格式和 API 版本策略
- 配置速率限制和 API 安全最佳实践

## 用法

- `/ecc:api-design` - 设计新的 API 端点或审查现有 API 契约

## 适用场景

- 设计新的 REST API 端点
- 审查现有 API 契约的规范性
- 添加分页、过滤或排序功能
- 实现 API 错误处理
- 规划 API 版本策略
- 构建面向公众或合作伙伴的 API
