# /ecc:santa-method

多代理对抗性验证框架：双独立审查+收敛循环，两者均通过才放行。

---

## 功能

- 双独立审查：两个无共享上下文的审查代理并行评估同一输出
- 刚性裁决门：两位审查者必须同时通过才算 NICE，任一不通过触发修复循环
- 收敛循环：收集问题 -> 修复 -> 重新审查，最多迭代 3 次后升级人工处理
- 批量采样模式：大规模批次时通过分层抽样降低成本，捕获系统性模式
- 审查度量：跟踪首次通过率、平均收敛轮次、问题分类和逃逸率

## 用法

- `/ecc:santa-method` - 在需要发布的输出、代码或内容审查前激活

## 适用场景

- 输出将被发布、部署或终端用户消费时
- 合规、法务或品牌约束必须强制执行时
- 代码在无人审查的情况下直接上线
- 批量内容生成中系统性错误难以通过抽查发现时
- 幻觉风险较高的场景（声明、统计、API 引用、法律语言）
