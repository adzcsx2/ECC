# /ecc:council

召集四角色评审团，为模糊决策提供结构化的不同意见。

---

## 功能

- 同时启动 Skeptic、Pragmatist、Critic 三个子代理作为独立评审角色
- 当前 Claude 会话担任 Architect 角色，提供第四个视角
- 防锚定机制：子代理只接收问题与上下文，不接收完整对话历史
- 输出结构化裁决，包含共识点、最强烈分歧和前提检查

## 用法

- `/ecc:council` - 当存在多个可行路径且无明显赢家时，获得多视角结构化的决策建议

## 适用场景

- 存在多个可信方案且需要权衡取舍的决策
- go/no-go 类型的发布或架构决策
- 需要第二意见或希望避免对话锚定偏差的场景
