# /ecc:regex-vs-llm-structured-text

结构化文本解析的决策框架：先用正则处理 95-98% 的场景，仅在低置信度边缘情况使用 LLM。

---

## 功能

- 决策树：文本格式一致性判断 -> 正则解析 -> 置信度评分 -> LLM 验证（仅边缘情况）
- 混合管线架构：Regex Parser -> Text Cleaner -> Confidence Scorer -> LLM Validator
- 置信度评分系统：自动标记缺失答案、选项不足、文本过短等低置信项
- 成本优化：实际案例中正则成功率 98%，LLM 调用仅需 ~5 次，相比全 LLM 方案节省约 95% 成本
- 不可变性模式：解析/清洗/验证步骤始终返回新实例

## 用法

- `/ecc:regex-vs-llm-structured-text` - 解析结构化文本（题库、表单、发票、文档）时使用
- 先尝试正则，仅对置信度 <95% 的项目调用 LLM

## 适用场景

- 题库/试题解析
- 表单数据提取
- 发票/收据处理
- 文档结构解析（标题、章节、表格）
- 任何有重复模式且对成本敏感的结构化文本处理
