# /ecc:product-lens

在动手构建之前验证"为什么"，运行产品诊断、创始人审查、用户旅程审计和功能优先级排序。

---

## 功能

- 产品诊断模式：自问 7 个关键问题，输出产品简评和建议
- 创始人审查模式：分析项目现状，评估 PMF 信号
- 用户旅程审计：模拟新用户体验，发现摩擦点
- 功能优先级排序：ICE 评分法（影响 x 信心 / 投入）排名
- 所有模式输出可执行文档，附带具体下一步

## 用法

- `/ecc:product-lens` - 在开始任何功能前启动产品诊断
- 可配合 `/browser-qa` 验证用户旅程审计结果
- 确定为"是，构建它"后，下一步转入 `product-capability`

## 适用场景

- 开始任何功能前验证方向
- 周度产品审查时确认是否在构建正确的东西
- 在多个功能之间难以决策时
- 发布前的用户旅程健康检查
- 将模糊想法转化为正式的产品概要
