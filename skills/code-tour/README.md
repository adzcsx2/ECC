# /ecc:code-tour

创建面向特定角色的 `.tour` 代码导览文件，用真实文件和行号锚定逐步走读。

---

## 功能

- 八种角色模板：新入职者、快速浏览、架构师、PR 审查员、RCA 调查员、安全审查等
- 五种步骤类型：内容说明、目录导航、文件+行号、选区锚定、模式匹配
- SMIG 写作规则：每个步骤描述场景、机制、影响和易错点
- 确定性文件路径：以 `.tours/<角色>-<焦点>.tour` 命名，可读可追溯
- 验证保证：每个文件路径和行号必须真实存在，不允许猜测

## 用法

- `/ecc:code-tour` - 根据需求生成面向特定读者的代码仓库导览文件

## 适用场景

- 为新维护者创建入门导览
- 为某个服务或包创建架构走读
- 围绕 PR 变更文件创建审查导览
- 为故障分析创建失败路径的追踪导览
